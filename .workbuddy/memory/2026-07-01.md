## P0 问题修复（2026-07-02）

修复了全部 3 个 P0 级技术债：

### P0-3: 事件循环反模式修复
- **文件**: `app/routes_quote.py` — `_process_single_file_sync`
- **变更**: `new_event_loop()` + `run_until_complete()` + `close()` → `asyncio.run()`
- **影响**: 正确取消残留任务、关闭传输、清理资源

### P0-2: _safe_add_column 修复 + Alembic 迁移
- **文件**: `app/database.py` — `_safe_add_column`
- **变更 1**: `except: pass` → 区分 "duplicate column"（DEBUG）和真正错误（WARNING）
- **变更 2**: 发现并修复隐藏 bug：SQLAlchemy 2.0 要求 `text()` 包装 SQL 字符串，原代码 `conn.execute(f"...")` 一直在静默失败
- **变更 3**: 创建 Alembic 迁移 `a1b2c3d4e5f6_timestamp_types.py`，使用 SQLite batch mode

### P0-1: 时间戳 String → DateTime 迁移
- **文件**: `app/db.py`, `app/models_orm.py`
- **策略**: 创建 `UTCDateTime` TypeDecorator（写入兼容 ISO 字符串，读取返回 datetime）和 `UnixTimestamp` TypeDecorator（写入兼容数字字符串，读取返回 float）
- **变更**: 25 个 ISO datetime 字段 → UTCDateTime，7 个 Unix 时间戳字段 → UnixTimestamp
- **优势**: 无需修改 35+ 处写入代码（`.isoformat()` 仍可用），读取端 FastAPI 自动序列化 datetime
- **更新文件**: `routes_admin.py`（审计清理用 datetime 对象）、`routes_quote.py`（CSV 导出处理 datetime）

### 验证结果
- 所有自定义类型单元测试通过
- DB 往返测试通过：`User.created_at` 返回 `datetime`，`AuditEvent.created_at` 返回 `datetime`
- `init_db()` 无错误（修复 `text()` 后 ALTER TABLE 正常执行）
