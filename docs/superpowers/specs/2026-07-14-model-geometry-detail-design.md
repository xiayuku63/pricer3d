# 模型几何信息详情设计

## 目标

在报价结果的展开详情中，将模型的包裹尺寸、表面积和体积汇总为一条独立的模型尺寸信息行，放置在“切片参数”标题下方、切片参数网格之前。体积信息需要加粗强调。

## 数据来源

报价结果项已包含以下由后端几何计算生成的字段：

- `dimensions`：包裹尺寸文本，单位为 `mm`。
- `surface_area_cm2`：表面积，单位为 `cm2`。
- `volume_cm3`：体积，单位为 `cm3`。

本次仅调整前端展示，不改变报价 API、数据模型或几何计算逻辑。

## 界面行为

桌面端结果表展开详情与移动端结果卡详情均使用相同的模型尺寸信息行。信息行位于现有“切片参数”卡片内部，格式为：

`模型尺寸  包裹 {dimensions} | 表面积 {surface_area_cm2} cm2 | 体积 {volume_cm3} cm3`

“体积 {volume_cm3} cm3”整体使用较高字重，以区别于其他几何信息。各字段保留现有后端返回的数值精度和单位。

若当前结果没有 `gcode_summary`，页面不创建“切片参数”卡片，因此也不展示该信息行；这与当前详情展示逻辑一致。若某一几何字段为空，仍保留可用字段，避免显示 `undefined` 或 `null`。

## 实现结构

在 `static/js/modules/quote-render.js` 中新增一个私有 HTML 构建函数，用于根据单个报价结果生成模型尺寸信息行。桌面和移动端的切片详情均调用该函数，再调用既有 `_buildGcodeDetailHtml()` 渲染切片参数网格。

该函数负责：

- 对模型字段进行 HTML 转义。
- 只输出有值的几何项。
- 对体积文本输出加粗样式。
- 在无任何可显示字段时返回空字符串。

## 验证

为这一私有渲染逻辑增加可导出的最小测试入口，使用 Node 内置测试框架验证：

- 包裹尺寸、表面积、体积及其单位均被渲染。
- 体积使用加粗样式。
- 缺失字段不会输出无效文本。
- 桌面和移动端切片详情均通过同一函数获得该行。

继续执行现有 `npm test`（如已有脚本）或通过 `node --test tests_js` 运行前端测试。
